## 工程规则

### git 操作相关


```bash
1. 先同步主分支

cd /513 (项目路径)
git status
git checkout main
git pull origin main

2. 新建分支
git checkout -b feat/your-feature-name (example)

3. 看这次改了什么
git status
git diff --stat

4. 提交改动
git add Version_?
git commit -m "Your Commit for this Modify"
commit 记得写做了什么内容

5. 推送分支
git push -u origin feat/your-feature-name

6. 创建 PR
推完以后 GitHub 的branch中会出现你们推的branch, 右边三个小点可以创建新的PR, 每一次提PR的时候一定写一个md, 说明这个branch做了哪些修改.

每个 feature 单独建一个 branch => 做完之后 PR 到主分支 => issue的右边找到development关联创建的branch

分支命名：

- `feat/xxx`：新功能
   例如：`feat/building-search`
- `fix/xxx`：修 bug
   例如：`fix/login-validation`
- `chore/xxx`：杂项、配置、依赖、脚本
   例如：`chore/init-seed-script`
- `refactor/xxx`：不改功能的重构
   例如：`refactor/review-service`
- `docs/xxx`：文档
   例如：`docs/update-readme`


```

### 当前目录框架

- `Final/Data` 是共享数据底座
- `Final/Version_A`、`Final/Version_B`、`Final/Version_C` 是三个人各自的完整版本线
- 每个版本内部按 `EDA / Experiments / Results / Notes` 组织
- 不要在个人版本里重复做整套数据清洗
